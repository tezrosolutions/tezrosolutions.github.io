Staging Environment

Website and Landing Pages

All the website and landing pages have to follow the staging workflow. This means whenever a change is requested on a page which is already live, make sure a staging equivalent is created at the top of it. This staging version will be used for any new development work and client demonstration.

If a new page is being developed, even then make sure that it follows the staging workflow.

The practice of creating live drafts will suspended.


Blogs

Also make sure you select a demo domain for all staging blogs. Delete the test blog when it is no longer needed.
